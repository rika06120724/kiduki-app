## 1. 概要（Overview）

シニア犬の日々の生活記録から、認知機能低下（CCD: Canine Cognitive Dysfunction）の兆候につながる「いつもと違う変化」を検知し、飼い主の早期の気づきを支援するWebアプリ。

健康管理全般ではなく、**犬の認知機能低下の早期気づきに特化**する。その子自身の生活パターンをもとに変化を検知し、受診や生活改善を検討するきっかけを継続的に提供する。

---

## 2. 背景・課題（Problem Statement）

### 2-1. 本質的な課題

飼い主は日々愛犬と接しているからこそ、緩やかに進行する変化を「いつもと同じ」という感覚に埋もれさせてしまう。日々の記憶は曖昧で「先週と比べてどうか」を正確に比較する手段がなく、「気のせいかもしれない」と判断を先送りしてしまう。結果として変化が進行してから気づくことになり、受診や対応が遅れる可能性がある。

> **本当に解決したい課題**：飼い主が日々の記憶や感覚だけでは、緩やかに進行する愛犬の変化を客観的に比較・判断できず、対応の機会を逃してしまうこと。

### 2-2. きっかけ

- 愛犬（チワワ）が9歳でシニア期に入り、加齢性疾患リスクを意識するようになった
- 過去に愛犬がてんかん発作を経験し、事前に変化を察知できなかった
- 職業訓練の睡眠アンケートで「高齢犬の夜泣き」の悩みが見られた
- 家族の認知症経験から、認知機能低下は本人だけでなく周囲の生活にも影響することを実感

---

## 3. ターゲットユーザー（Target User）

| 観点 | 内容 |
| --- | --- |
| 年齢 | 20代～50代 |
| ペット状況 | シニア期に入り始めた犬を飼っている |
| 行動特性 | 愛犬と毎日一緒にいるため、動きが減った・夜鳴きが増えた・食いつきが鈍くなったといった変化が徐々に進行し、明確な異変として認識しづらい。「気のせいかも」と自分に言い聞かせ、記録や確認をしないまま過ごしている |
| 環境 | Webブラウザからアクセスできるデジタルデバイス（PC・スマホ・タブレット）を利用でき、朝晩にペットの様子を観察できる家庭 |

### 想定シナリオ（利用開始のきっかけ）

「あれ、最近こうだったかも」と思いつつ確証が持てず流してしまった経験、または後から「そういえば前から変だったかも」と気づいて後悔した経験を持つ飼い主が、愛犬のプロフィール登録と「気のせいかも」と思った日の記録を最初の一歩として利用を開始する。

---

## 4. 提供価値（Value Proposition）

シニア犬の緩やかな認知機能の変化を気にかける飼い主が、毎日の記録を登録するだけで、経験や感覚だけに頼らず、アプリが示す「いつもと違う」変化をもとに受診や経過観察を判断できるようになるサービス。

### ユーザーの変化

- **行動の変化**：記録するだけでなく、蓄積データから「いつもと違う変化」を確認する習慣が身につく
- **気持ちの変化**：漠然とした不安から、データに基づいて変化を把握できる安心感へ
- **判断の変化**：経験や感覚ではなく、日々のデータに基づいて受診や経過観察を検討できる

---

## 5. 競合・差別化（Competitive Landscape）

| サービス | 主な機能 | 認知機能低下への対応 |
| --- | --- | --- |
| ペットケア（通院・投薬・体調記録） | 5件法の体調ログ、年間ヘルスレポート、6大指標チャート、お世話リマインダー | 排便のみ異常検知（固定閾値による判定と推測、他5指標は非対応） |
| ペットノートPlus | 自由記録の健康記録、リマインダー、支出管理、日記、カレンダー、チャート | 異常検知・アラート機能なし |

### 差別化ポイント

- 既存サービスは記録・履歴管理が中心で、認知症の兆候に着目した変化分析機能が乏しい
- 唯一の異常検知アプリも対応項目が「排便」のみで、他の指標（食欲・飲水量・活動量・異常行動）には反応せず、長期パターン学習ではなく固定閾値判定と推測される
- 本サービスは特定項目に限定せず、複数指標を横断的に見て**その子自身の過去パターンとの差分**から「いつもと違う」を検知する

## 6. スコープ（Scope）

### 6-1. MVPで作る機能

#### ① ユーザー登録・ログイン
- メールアドレスとパスワードでユーザー登録・ログインできる

#### ② 日々の生活記録
以下7項目を5件法（5段階選択肢）で記録する。犬の認知機能評価指標 **DISHAA** を参考に、認知症との関連性が高い特徴量として選定。

1. 今日、目的なく歩き回る・同じ場所をぐるぐる回る様子はありましたか？
2. 今日、あなたの姿を見たときの反応はどうでしたか？
3. 今日、名前を呼んだときの反応はどうでしたか？
4. 昨夜、落ち着かない様子や夜中に起きている様子はありましたか？
5. 今日、活動量はいつもと比べてどうでしたか？
6. 今日、トイレを失敗しましたか？
7. 今日の食欲はどうでしたか？

#### ③ 変化検知アルゴリズム
- 利用開始直後はダミーデータを基にしたモデルで暫定的な判定を行い、記録が一定期間蓄積した後、その記録を反映したモデルへ更新する
- Isolation Forestを用いて、複数の生活記録項目を組み合わせた行動パターンから「その子にとって普段と異なる状態」を異常として検知する
- 異常検知時は各特徴量のロバストZスコアを算出し、どの項目が普段と比べて大きく変化したかを表示する

#### ④ 変化スコア表示
- Isolation Forestによる異常スコアを表示
- ロバストZスコアを用いて各項目が通常時からどの程度変化したかを可視化
- 過去の記録と比較し、現在の状態が通常時からどの程度変化しているかを視覚的に確認できる

#### ⑤ 変化アラート表示
- 一定以上の異常が検知された場合、「いつもと違う変化」の発生を通知する
- 認知症を診断するものではなく、受診や経過観察を検討するきっかけを提供する

#### ⑥ ペット情報の管理
- ペットのプロフィール（名前・年齢・性別・種類・アイコン）を登録・編集できる

#### ⑦ カレンダー閲覧
- 日々の記録と検知結果をカレンダー形式で閲覧できる

#### ⑧ 過去日の記録
- 記録し忘れた日について、過去の日付を指定して遡って記録できる
- 既に記録がある日は上書きして修正できる

### 6-2. 本リリースで作る機能（MVP後）

- 気になる様子を自由記述で記録できるメモ機能
- メモ内容を自然言語処理によって解析し、頻出する症状や特徴語を抽出
- 多頭飼いへの対応（複数の犬を登録可能に）

### 6-3. スコープ外（Out of Scope）

- 犬の認知症の医学的診断
- 犬以外のペット種への対応（将来検討）
- ネイティブアプリ（Webアプリのみ）

## 7. 非機能要件・技術スタック

| 項目 | 選定 |
| --- | --- |
| フレームワーク | Django |
| DB | PostgreSQL |
| デプロイ先 | Render（GitHub連携による自動デプロイ） |
| 異常検知 | scikit-learn（Isolation Forest） |
| データ加工 | pandas |
| 数値計算 | NumPy |
| ロバストZスコア算出 | SciPy |

### 技術選定理由

- **Django**：認証・管理画面が標準装備のため、異常検知機能の実装に注力できる
- **PostgreSQL**：時系列データの管理・集計に強く、将来的な分析・拡張も見据えて選定
- **Render**：GitHub連携による自動デプロイが可能で、Django・PostgreSQL構成をシンプルに構築できる
- **scikit-learn / Isolation Forest**：正常・異常のラベル付きデータの用意が困難なため教師なし異常検知を採用

### 将来検討（MVP不採用）

- **FastAPI**：まずは一つのフレームワークでMVP完成を優先するため不採用。本リリース後、API分離が必要になった段階で検討
- **React**：MVPではDjangoテンプレートを優先。本リリース後、フロントエンド・バックエンド分離が必要になった段階で検討

## 8. リスクと対策（Risks & Mitigations）

### リスク①：初期データ不足による異常検知精度の不安定さ
- **懸念**：Isolation Forestは通常時の行動パターン学習に基づくため、データ量が不足すると判断が安定しにくい
- **対策**：利用開始直後はダミーデータによる暫定判定を行い、一定期間データが蓄積した後にその犬自身の記録を反映した判定へ移行する

### リスク②：入力負担による継続利用の離脱
- **懸念**：異常検知の精度向上には継続的なデータ蓄積が必要だが、入力負担が大きいと記録が途切れやすい
- **対策**：入力を5件法（5段階選択）中心とし、短時間で記録できるUIを採用する

### リスク③：Isolation Forestの実装難易度
- **懸念**：初めて扱うアルゴリズムであり、検証・評価方法も学習しながら進める必要がある
- **対策**：MVPではIsolation Forestによる異常検知を目標とし、想定以上に時間を要する場合はロバストZスコアによる閾値ベースの判定を代替手段として実装する。ダミーデータや疑似的な緩やかな悪化パターンを含むシナリオデータで検知タイミングを検証・改善する
